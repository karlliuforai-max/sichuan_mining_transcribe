## 1. 用户当前目标

用户要构建一个可跨平台部署、可被不同 agent 调用的四川矿业会议转写后处理 skill。

典型使用场景：

- macOS 或 Windows 上用 Codex/Claude Code 调用 skill。
- 飞书连接 OpenClaw agent，通过 `/skill` 调用处理会议转写，这是最高频场景。
- 未来可能部署到阿里云服务器。

核心诉求：

- 对四川话、乐山话、普通话混杂的矿业会议转写进行纠错。
- 重点处理锂矿、选矿、采矿、品位、回收率、精粉、尾矿、运输、报价、付款、资质等专业内容。
- 输出可信、可跟进、可复核的会议纪要，而不是逐字稿。
- 长任务要实时显示进度，降低等待焦虑。
- skill 要能随经验积累知识，但不允许模型把猜测直接写入正式错词表。

## 6. DeepSeek 当前设计

默认 DeepSeek 模式已成为生产质量默认，因为用户认可其纪要质量明显高于本地模式。

当前模型分层：

- `--deepseek-tier flash`
  - 模型：`deepseek-v4-flash`
  - thinking：默认 disabled
  - 用途：日常默认、速度优先

- `--deepseek-tier pro`
  - 模型：`deepseek-v4-pro`
  - thinking：默认 enabled
  - `reasoning_effort: high`
  - 用途：高质量、复杂转写、正式归档

官方文档参考：

- https://api-docs.deepseek.com/zh-cn/
- https://api-docs.deepseek.com/zh-cn/guides/json_mode

注意：

- 代码中已按 JSON Output 配置 `response_format: {"type": "json_object"}`。
- Base URL 默认为 `https://api.deepseek.com`，代码自动补 `/chat/completions`。
- 当前没有把 key 写入任何项目文件。
