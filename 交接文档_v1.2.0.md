## 5. DeepSeek 复测结果

测试文件：

```text
转写源/05-24 客户会议_ 锂矿合作洽谈.pdf
```

### Flash v2

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-flash-0524-v2/
```

结果：

- 模型：`deepseek-v4-flash`
- thinking：disabled
- 总耗时：约 167.8 秒
- 纠错全文：13,519 字
- 会议纪要：2,238 字
- 结论：成功，未扩写，适合日常默认。

重要对比：

- 旧 flash 的 `03_corrected.md` 为 23,295 字，明显扩写。
- v2 的 `03_corrected.md` 为 13,519 字，已解决扩写问题。

### Pro + Thinking Enabled

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-pro-0524-v2/
```

结果：

- 模型：`deepseek-v4-pro`
- thinking：enabled
- 纠错全文：83 字
- 结论：不可用。返回空 `corrected_text`，不适合作为默认或推荐模式。

### Pro + Thinking Disabled

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-pro-no-thinking-0524-v2/
```

结果：

- 模型：`deepseek-v4-pro`
- thinking：disabled
- 总耗时：约 327.8 秒
- 纠错全文：13,491 字
- 会议纪要：3,032 字
- 结论：可用，质量更细，但耗时约为 flash 的 2 倍，适合重要会议。

## 6. 当前推荐使用方式

日常默认：

```bash
python3 sichuan-mining-transcribe/scripts/transcribe_pipeline.py \
  --input "转写源/05-24 客户会议_ 锂矿合作洽谈.pdf" \
  --output sichuan-mining-transcribe/outputs/run-flash \
  --mode deepseek \
  --deepseek-tier flash \
  --progress text \
  --knowledge-dir sichuan-mining-transcribe/knowledge \
  --learn-policy candidate
```

重要会议：

```bash
python3 sichuan-mining-transcribe/scripts/transcribe_pipeline.py \
  --input "转写源/05-24 客户会议_ 锂矿合作洽谈.pdf" \
  --output sichuan-mining-transcribe/outputs/run-pro \
  --mode deepseek \
  --deepseek-tier pro \
  --deepseek-thinking disabled \
  --progress text \
  --knowledge-dir sichuan-mining-transcribe/knowledge \
  --learn-policy candidate
```

无网络/隐私兜底：

```bash
python3 sichuan-mining-transcribe/scripts/transcribe_pipeline.py \
  --input "转写源/05-24 客户会议_ 锂矿合作洽谈.pdf" \
  --output sichuan-mining-transcribe/outputs/run-local \
  --mode local \
  --progress text
```
