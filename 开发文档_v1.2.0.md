# 四川矿业会议转写纠错与纪要生成 Skill 开发文档 v1.2.0

版本日期：2026-05-28

本文档是在 `开发文档_v1.1.0.md` 基础上的测试与策略更新版。原始 `开发文档.md` 和 `交接文档.md` 是开发前 v1.0.0 基线，仍按用户要求保持不动。

## 1. 本版本更新摘要

本轮完成了 DeepSeek API 复测、`.env` 自动加载、请求稳定性增强和默认策略调整。

主要结论：

- `deepseek-v4-flash` 在新 prompt 下已经解决纠错扩写问题，可作为日常默认。
- `deepseek-v4-pro + thinking enabled` 当前不适合本项目的 JSON 纠错流程，表现为非常慢且返回空正文。
- `deepseek-v4-pro + thinking disabled` 可稳定输出，纪要更细，但耗时约为 flash 的 2 倍，适合作为重要会议/高要求模式。
- 当前默认策略调整为：flash 默认；pro 默认也关闭 thinking；thinking enabled 仅保留为实验参数。

## 4. DeepSeek 稳定性增强

本轮对 `sichuan-mining-transcribe/scripts/llm_correct.py` 做了两项生产化增强。

### 4.1 请求重试

pro 测试中曾遇到：

```text
http.client.IncompleteRead
```

这是长请求/分块响应常见瞬断。当前已新增 `post_with_retries()`：

- 默认最多 3 次请求。
- 捕获 `TimeoutError`、`IncompleteRead`、`HTTPException`、`URLError`。
- 失败后指数退避等待。
- 多次失败后抛出明确错误。

### 4.2 空 corrected_text 防护

`deepseek-v4-pro + thinking enabled` 曾返回空 `corrected_text`，导致最终 `03_corrected.md` 只有 83 字，`04_minutes.md` 也为空洞。

当前已增加防护：

- 如果某段模型返回空 `corrected_text`，自动回退为本地纠错后的原 chunk。
- 同时写入 `uncertain_items`，提醒该段模型返回结构异常。

这能避免以后因为单段模型异常导致整份会议纪要空掉。

## 5. 复测结果

测试样本：

```text
转写源/05-24 客户会议_ 锂矿合作洽谈.pdf
```

### 5.1 Flash v2

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-flash-0524-v2/
```

结果：

| 指标 | 结果 |
|---|---|
| 模型 | `deepseek-v4-flash` |
| thinking | disabled |
| 总耗时 | 约 167.8 秒 |
| 纠错阶段 | 约 149.5 秒 |
| 语义纪要阶段 | 约 18.4 秒 |
| 清洗文本长度 | 13,967 字 |
| 纠错全文长度 | 13,519 字 |
| 纪要长度 | 2,238 字 |

判断：

- 新 prompt 已解决旧 flash 版本的扩写问题。
- 旧 flash 的 `03_corrected.md` 曾达到 23,295 字；v2 降至 13,519 字，与原文长度相当。
- 纪要结构完整，适合日常默认。

### 5.2 Pro + Thinking Enabled

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-pro-0524-v2/
```

结果：

| 指标 | 结果 |
|---|---|
| 模型 | `deepseek-v4-pro` |
| thinking | enabled |
| 结果 | 不可用 |
| 纠错全文长度 | 83 字 |

判断：

- 该组合非常慢。
- 返回空 `corrected_text`，导致产物无效。
- 当前不应作为默认或推荐模式。
- 仅保留为实验参数。

### 5.3 Pro + Thinking Disabled

输出目录：

```text
sichuan-mining-transcribe/outputs/sample-deepseek-pro-no-thinking-0524-v2/
```

结果：

| 指标 | 结果 |
|---|---|
| 模型 | `deepseek-v4-pro` |
| thinking | disabled |
| 总耗时 | 约 327.8 秒 |
| 纠错阶段 | 约 275.9 秒 |
| 语义纪要阶段 | 约 51.9 秒 |
| 纠错全文长度 | 13,491 字 |
| 纪要长度 | 3,032 字 |

判断：

- 输出稳定，没有扩写。
- 纪要比 flash 更细，对关键数据、风险和商务条件提取更充分。
- 耗时约为 flash 的 2 倍，适合重要会议、高价值客户会议、正式归档，不适合日常默认。

## 6. 推荐使用策略

当前建议固定三档：

| 场景 | 推荐模式 |
|---|---|
| 日常会议 | `--mode deepseek --deepseek-tier flash` |
| 重要会议/正式归档 | `--mode deepseek --deepseek-tier pro --deepseek-thinking disabled` |
| 离线/隐私敏感/无 key | `--mode local` 或 `--mode auto` |

不建议：

```bash
--deepseek-thinking enabled
```

除非明确是在做模型能力实验。
